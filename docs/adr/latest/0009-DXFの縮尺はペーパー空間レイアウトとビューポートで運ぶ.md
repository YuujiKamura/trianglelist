# 0009: DXF の縮尺はペーパー空間レイアウト + ビューポートで運ぶ

- 日付: 2026-06-12
- 状態: 採用 (DxfFileWriter に実装・実機検証済)
- 起点: user「以前から疑問だったのが、DXF 図面として縮尺を正しく設定できないこと。
  デフォルトだと A3 で 50 分の 1 だけど、これを正しくデータに仕込むのが、どうも無理っぽい
  って事になった時があった」→ 調査の結果「無理」は条件付きで覆り、「やってみて」で実装。

## 診断 — DXF モデル空間は縮尺を持てない、運ぶのはペーパー空間

DXF のモデル空間は無次元で「この図面は 1/50」を書く正規の場所が無い (DWG→DXF 変換で
縮尺が壊れるのは業界の既知問題)。Jw_cad は読み込み時に図面の外形サイズから縮尺を逆算する
ヒューリスティックで対処し、国交省 CAD 製図基準が納品形式に SXF(P21) を要求するのも
DXF が縮尺を運べないため。だが **AutoCAD R2000 (AC1015) のペーパー空間 (レイアウト) には
ビューポートの尺度という正規の置き場がある**。CADWe'll 土木はこれを読む。

## 決定 — ペーパー空間に LAYOUT + VIEWPORT 2 枚を書く

実装 (`common/datamanager/DxfFileWriter.kt` の `writePaperSpaceViewport` / ObjectsBuilder
の LAYOUT/PLOTSETTINGS)。ezdxf 1.4.1 が生成する正しい DXF を実測 (TagCollector dump /
recover) して group code 列を合わせた:

1. **VIEWPORT は 2 枚必須** — AutoCAD R2000 のレイアウトは「メインビューポート
   (id=1, status=1, 紙面全体)」が必須で、これを欠くと CADWe'll はビューポートシートを
   構築せず「モデル/ベース」しか出ず尺度が伝わらない。id=1 (紙面全体、ezdxf 既定の
   中心=紙×25・大きさ=紙×55) + id=2 (図面を 1/分母 で映す内容) を書く。倍率は
   group 41 (紙上の高さ mm) / 45 (モデル空間の表示高さ mm) = 1/分母 で表現。
2. **PLOTSETTINGS / LAYOUT に owner handle (group 330) を付ける** — ハンドル直後の
   owner は PLOTSETTINGS → ACAD_PLOTSETTINGS 辞書、LAYOUT → ACAD_LAYOUT 辞書を指す
   (AcDbLayout 内のブロックレコード参照 330 とは別階層)。欠けていると ezdxf が
   「invalid owner handle」として黙って自動補正していた。
3. **LAYER ハンドルの一意性** — TablesBuilder/DxfTable の LAYER テーブルで C-COL-COL1 と
   C-TTL-FRAM が両方ハンドル 74 を持つコピペバグを修正 (C-TTL-FRAM → 75)。回帰テスト
   `writeDxf_entityHandles_areUnique` で TABLES 以降の group code 5 重複を禁止。

縮尺の出所は `currentPrintScale`、分母は `printscale_ * 100` (getPrintScale が図形サイズで
0.5→1/50, 2.0→1/200, 5.0→1/500 と自動選択)。インポートダイアログの縮尺欄はモデル変換
縮尺で別物 (マニュアルいわく手入力前提、外形から自動推定) ── DXF の尺度はレイアウト経由で
伝わるのであって、あの欄に入るわけではない。

## 縮尺と用紙サイズの可変性 — 両方とも可変 (2026-06-12 用紙 SoT 統合で完了)

- **縮尺は完全可変** — ビューポートの 45 (= 紙高 × den) も PLOTSETTINGS の 142/143
  (= 1/den) も `den` で動く。1/50・1/200・1/500 すべて検証済。
- **用紙サイズも可変、出所は `paper` 1 つ** — かつては Paper / DxfPaper / DxfConstants と
  枠ジオメトリのハードコード (420 vs 42000 vs 38×25) に散らばっていた。これを `DrawingFileWriter`
  の `paperWcm/paperHcm/paperName` に集約し、枠外形・上部タイトル・右下タイトル欄を
  「用紙端アンカー」(右端 = paperWcm-1、上端 = paperHcm-k、中央 = paperWcm/2) に書き換え。
  `DxfFileWriter.save()` が `paper` → paperWcm/Hcm/Name を伝播し、sizeX_/sizeY_・図形センター・
  ビューポート・PLOTSETTINGS・縮尺ラベルがすべて 1 点から追従する。Paper に A0-A4 横向き
  presets を追加。**A3 では従来と byte 同値** (既存 golden が再採取なしで pass)。表題欄は
  規格どおり右下固定サイズの隅アンカー、用紙を大きくすると作図領域だけ広がる。
  - 残: getPrintScale の縮尺フィット域 (38×25cm) は model 層にあり paper を知らないため
    A3 基準のまま。大きい用紙では余白が増えるだけで実害は無いが、フィット域も paper 駆動に
    するのは将来段。web キャンバスの枠 (WebFrame.kt) も画面専用で A3 固定 (DXF 出力とは別系統)。
