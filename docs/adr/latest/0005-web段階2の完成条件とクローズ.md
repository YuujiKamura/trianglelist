# ADR 0005: Web 段階2 の完成条件を定義し、ここで閉じる

- 日付: 2026-06-10
- 状態: 採択 — 完成条件は 2e (ea56352) 時点で達成済み、段階2 はクローズ
- 関連: ADR 0003 (寸法 2 層位置決め)、insights #75 (robolectric-launch-smoke) / #76 (wasm wrapper stale)

## Context

Web 移植 (段階2a〜2e) は app の機能を順に埋める形で進んだが、「どこまで行ったら移植できたと言えるか」の終わりが言語化されておらず、2f、2g…と際限なく足せる構造だった (2026-06-10 yuuji「やりたいことを沢山言って、とりとめのない感じになっている」への応答として、終わりの定義を文書に固定する)。

## Decision

**完成条件 (1 行)**: app の基本動線 ── 入力・接続・寸法操作・書き出し・保存 ── が web で全部でき、出力が app と golden 一致すること。

この条件は以下で達成済み。段階2 はここで閉じ、以降の機能追加は新しい段階番号 (または ADR) で起こす。

| 動線 | commit | 内容 |
|---|---|---|
| 入力 (表編集) | 4eee826 (2a) | 編集・追加・削除→即再描画 |
| 書き出し | 94a6a64 (2b) | CSV/DXF/SFC、golden 同値テストで app と同一出力を固定 |
| 操作系 | 067f600 (2c) | ズーム・パン・タップ選択 (isCollide を共有) |
| 入力 (実動線) + 接続 + 保存 | e9da82f (2d) | 3行フォーム+FAB+タップ親指定+接続追加+undo+autosave |
| 寸法操作 | ea56352 (2e) | W/H フリップ+旗揚げ+番号サークル移動 (ADR 0003 式⊕override 二層) |
| 安全網 | bfe262d / b18f6be | 見た目 golden 2層 + エミュレータなし E2E (Robolectric+Roborazzi) |

## 残ギャップ (= 段階2 の外、着手は別途判断)

app の操作一覧 (app/src/main/res/layout/ の fab id 全数 grep、2026-06-10) と web/index.html の照合:

- **出力**: fab_xlsx / fab_pdf — web は CSV/DXF/SFC のみ
- **図形操作**: fab_rot_l / fab_rot_r (回転)、fab_deduction (控除)、fab_numreverse (番号正逆)、fab_nijyuualign
- **見た目**: fab_fillcolor (塗り色)、fab_texplus / fab_texminus (文字サイズ)
- **共有**: fab_share / fab_mail (web では別概念になる)
- 対象外 (dev 用): fab_debug / fab_testbasic / fab_dxfview
- 既知の質の課題: 旗揚げテキストが初期 fit の範囲外に出ることがある → LabelPlacer (ADR 0002/0003 の段階3) のスコープ

## Consequences

- 「Web 移植できた」の判定基準が文書に固定され、以降の追加要望はこの ADR を起点に新段階として切れる
- 次の幹の候補は 3 つ: (a) 残ギャップから選んで段階2f、(b) LabelPlacer (段階3、自動配置)、(c) ADR 0003 残段階 (Triangle の dimPoint キャッシュ引き剥がし)。どれを取るかは yuuji の判断待ちで、それまで新機能は積まない
